Program ——

From HLL

Produce a Object Module

Assembler (or compiler) translates program into machine instructions Provides information for building a complete program from the pieces

  • Header : described contents of object module
  • Text segment : translated instructions
  • Static data segment : data allocated for the life of the program
  • Relocation info : for contents that depend on absolute location of loaded program
  • Symbol table : global definitions and external refs
  • Debug info : for associating with source code

Link Object Modules

Produces an executable image

    1. Merges segments
    1. Resolve labels (determine their addresses)
    1. Patch location dependent and external refs Could leave location dependencies for fixing by a relocating loader
  • But with virtual memory, no need to do this
  • Program can be loaded into absolute location in virtual memory space

Load a Program

Load from image file on disk into memory

    1. Read header to determine segment sizes
    1. Create virtual address space
    1. Copy text and initialized data into memory
    • Or set page table entries so they can be faulted in
    1. Set up arguments on stack
    1. Initialize registers (including $sp, $fp, $gp)
    1. Jump to startup routine
    • Copies arguments to $a0, … and calls main
    • When main returns, do exit syscall
